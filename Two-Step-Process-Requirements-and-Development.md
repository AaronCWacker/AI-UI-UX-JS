The two step process is my understanding of best way to do design and build competitively with AI.

The Design process attempts to use a prompt specification to describe an outcome that is a knowledge based outline.  This includes key approach of asking for output in markdown outline with emojis and keep it simple yet uncover the process, agentic opportunities to get assets and techniques or content from knowledge.  

Then the Development step attempts to produce a ready-to-run output so chooses the implementation stack (HTML/JS or Python) based on desired output runtime stack.

Minimization or First principles is used in theory in order to keep both vectorized pursuits seperate and prevent going off task.

This is aking to Invent (Design) and Build (Development) processes which require different inputs and outputs.

This is then used in parallel across a set of intelligent frontier models.  The model sets are then ranked by effectiveness and creativity as well as usefulness and reach of output.

Output which is never made public is never used hence little or no value.

Presentation of output in a computable external facing app is useful immediately as proof that the system works.  This functionally holds true even if just shared with communications.


For this particular experiment this required these two prompt specifications running on GPT (1st place), Grok (2nd place), Claude (3rd) and Gemini (4th) and was run 5/24/2026.

This ranking changes from experiment to experiement but is consistent with state of prompts and state of models on that date.

Gemini in this case when asked in development step to produce a html js app (something it normally did really well which should be 1st place) went another way due to a new feature that generates code for a visualization.

This was incompatible with the development request which clearly asks for Canvas preview of html js.  So while the visualization was visible - it has prioritized a new feature above the user request hence is last place.

The requirement in development step to produce the runnable asset is a necessary step inside a sandbox so that we can test and iterate while maintaining the memory of steps or specs in order which rpoduce content that can be revised quickly in an ideal testing inner loop which needs human intelligence to check on it and rate speed, usefulness,
